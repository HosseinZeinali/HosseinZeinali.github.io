Open File Deletion

November 8, 2020

In this article i declare “open file deletion” issue and solution.

Today i was checking my server’s disk usage, and found something unusual. The result i got from, du and df were not close to each other. These are the result i got for du, and df commands.

sudo du -sh /
18GB    /

sudo df –h /
Filesystem     1K-blocks     Used       Available Use% Mounted on
/dev/vda1      101445540     86231317   14234138  86%  /

du shows we have used 18GB of storage, in contrary df shows we have used nearly 86GB. I did some research, and found out df result is based on file system layer, and du result is based on file/directory layer. And this issue usually happens when we delete an open file, such as deleting a log file, when it is using by an application, and this is called “Open file deletion”. In order to solve this issue, We should first find the “deleted open file” process, then kill it. this will free up the occupied filesystem inode. This is the command to get open deleted files

lsof -nP | grep '(deleted)'

We should find large size files, from the result of lsof command. For example this is the process of “deleted open file” on my server:

php 23947 root 6w REG 252,1 66803307926 258226 /usr/share/nginx/html/var/log/prod.deprecations.log (deleted)

And then we will kill the process by this command. In my case the $PID was 23947.

sudo kill -9 $PID

That’s it.
