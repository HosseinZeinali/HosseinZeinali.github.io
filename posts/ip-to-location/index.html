ip to location

September 25, 2020

I have developed a tool which i call it ip2lo to make a database to map user ips to their location. Which can be used to recognize users coutryside by their ip address.

Why i have made ip2loc?

I couldn’t find any standalone opensource geolocating service, which is working, without using external apis, So I decided to make ip2loc. ip2loc updates the database automatically, archives the data, and also has a service based on http to get ip address details.

Database

We keep ip_start, ip_end, ip_count, status, cc in the database. ipv4 and ipv6 tables are separated and as we want to have archive the tables we make new tables and keep the old ones instead of updating it.

How does it work

This tool uses 5 Regional Internet Registrars (RIR) databases: AFRINIC, APNIC, ARIN, LACNIC and RIPE in order to make the database. So we need to have internet access to the below addresses to download the files:

  • https://ftp.afrinic.net/pub/stats/afrinic/delegated-afrinic-latest
  • https://ftp.lacnic.net/pub/stats/lacnic/delegated-lacnic-latest
  • https://ftp.arin.net/pub/stats/arin/delegated-arin-extended-latest
  • https://ftp.apnic.net/pub/stats/apnic/delegated-apnic-latest
  • https://ftp.ripe.net/ripe/stats/delegated-ripencc-latest

It checks the (RIR) database files checksum every 6 hours, if they have updated it downloads and parses the database files, transforms the data to our own data model and stores it to the database.

We also have http endpoint to get the ip information by a simple http call.

GET http://localhost:8082/api/ip/{ip}

You can get ip2loc from this repository on my github page:

https://github.com/HosseinZeinali/ip2loc
